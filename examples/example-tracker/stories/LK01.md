## Scope boundary

**In scope**

- Define the short-code generation and link storage boundary.
- Create the create-link entry surface.

**Out of scope**

- Redirect handling; see LK02.
- Click analytics; see LK03.

| Type | Item | Evidence or resolution path |

## Artifact boundaries

- PRD owns the Linkly what/why and acceptance criteria.
- Tracker owns sequencing, status, owner, plan, and PR fields.
- Story file owns story-local scope, citations (brief-level), and implementation design (plan-approved level).
- Implementation plan owns execution steps.
- Runtime artifacts own execution evidence.

## Canonical impact

Introduces the short-code generation algorithm and link store schema — both are architectural decisions that LK04 must fold into canonical docs.

## Candidate surfaces

- **Files/modules:** link store, short-code generator, create-link endpoint or command.
- **Queries/schema:** link record keyed by short code.
- **Prompts/tools:** none expected.
- **Events/metrics:** optional create-link audit event.
- **Components/routes:** create-link API or form surface.

## Validation expectations

- Creating a link returns a stable short code for a valid URL.
- Invalid URLs are rejected.
