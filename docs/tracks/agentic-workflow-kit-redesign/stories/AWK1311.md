## Scope boundary

**In scope**

- Unify the two vitest configs so coverage spans both `test/**` and `tests/**`; re-baseline the
  ratchet thresholds against the true combined number (no regression, still trending toward 90).
- Add one end-to-end story-run test: fake `StoryRunner` + real runner / journal / completion-gate /
  artifact store, asserting launch → verified-or-blocked outcome with artifacts written to disk.
- Add direct tests for the live `codex_reply` / `codex_interrupt` execution path (routed through the
  contract per AWK13.8), lifting `mcp/codexControl.ts` above its ~8% floor.

**Out of scope**

- Reaching the aspirational 90% target in one story (ratchet trends, not jumps).
- Rewriting existing suites; this story consolidates config and adds the missing high-value tests.
- Real Codex/GitHub network calls in the e2e test (use fakes at the driver/collaboration seams).

## Assumptions and blockers

| Type | Item | Evidence or resolution path |
| --- | --- | --- |

| Blocking question | Single merged config vs a workspace with one coverage report? | implement-next picks the layout that yields one combined number. |
