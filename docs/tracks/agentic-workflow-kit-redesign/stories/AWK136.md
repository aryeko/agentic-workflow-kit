# AWK13.6 test trust and coverage ratchet story brief

not implementation-ready; create a detailed technical story spec before plan/code

## PRD criteria

| Criterion | Product outcome |
| --- | --- |
| RUN-6 | Each recoverable stopped state has a test that fails if the behavior regresses. |
| HC-3 | The merge gate and GitHub verification have negative-path tests (fake/fabricated state is rejected). |
| Q-5 (quality bar) | Coverage is sufficient to independently trust time/outcome/verification metrics; the project coverage target is met. |

## Technical solution sections

| Section | Relevance |
| --- | --- |
| Release-hardening design: AWK13.6 | Defines the negative-path test set, end-to-end smoke, and coverage ratchet. |
| Testing strategy | The runtime/driver/analyzer test layers must cover the safety-critical negative paths. |

## Dependencies

| Dependency | Reason |
| --- | --- |
| AWK13.5 | Tests target the final, decomposed module shape so they do not churn during refactor. |

## Scope boundary

**In scope**

- Add negative-path tests: merge gate rejecting a non-ancestor merge
  (`isCommitReachableFromRef === false`); real-git adapter tests for `isCommitReachableFromRef` /
  `readFileFromRef` / `refreshBaseBranch` (temp repos); malformed NDJSON readers; concurrent
  tracker-claim race; operator-abort interrupting a live child; one test per RUN-6 recoverable state.
- Add an end-to-end story-run smoke against the fake driver covering
  implement → verify → (mocked) collaboration verification → completion.
- Ratchet `packages/orchestrator/vitest.config.ts` coverage thresholds upward toward 90 (existing
  `TODO`) and gate regressions.

**Out of scope**

- Production behavior changes (tests only; fixes belong to AWK13.1–13.5).
- A full repeatable evaluation/benchmark harness (FUT-1 keeps it out of V1).

## Assumptions and blockers

| Type | Item | Evidence or resolution path |
| --- | --- | --- |
| Assumption | Temp-repo git harness already exists and is the pattern to extend. | Present in `git-inspector.test.ts` / `child-workspace-preparer.test.ts`. |
| Assumption | Coverage can reach ~90 once AWK13.1–13.5 land; the ratchet rises in steps if needed. | Current 82.6%/72.8% baseline. |
| Blocking question | Which fakes need upgrading so they stop encoding the expected answer (e.g. `FakeGitInspector.isCommitReachableFromRef` always true)? | implement-next audits fakes that currently make negative paths unreachable. |

## Artifact boundaries

PRD owns outcomes. Release-hardening design owns the test plan. Tracker owns sequencing. This brief
owns local scope. The detailed spec owns the exact fixtures, fake upgrades, and threshold steps.

## Candidate surfaces

- **Files/modules:** `packages/orchestrator/tests/*`, `packages/orchestrator/vitest.config.ts`, test
  fixtures

- **Prompts/tools:** fake driver / mocked collaboration inspector
- **Events/metrics:** assertions on emitted events and verified evidence
- **Components/routes:** none

## Validation expectations

- Each named negative path has a failing-on-regression test.
- Coverage thresholds raised and met; CI runs the ratcheted gate.
- `pnpm check`
