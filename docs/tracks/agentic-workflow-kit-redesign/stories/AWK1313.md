## Scope boundary

**In scope**

- Replace `runner: unknown` + structural `as <…>Runner` casts at `runner/ChildSupervisor.ts:53`,
  `runner/ChildLaunchRecorder.ts:57`, and `runner/WorkflowRunnerEligible.ts:44` with a typed
  runner-context interface, breaking the `WorkflowRunner` cycle at the type layer.
- Decompose `executeChildWithSupervisor` (`runner/ChildSupervisor.ts:52`, ~270 lines) into named
  collaborators with the externally observable supervision behavior unchanged.

**Out of scope**

- Any change to supervision behavior (timeouts, abort propagation, recovery-guard wiring).
- Renaming host-named back-compat surfaces (`DEFAULT_ARTIFACT_ROOT_DIR`, `config.codex`) — intentional.
- Telemetry/budget changes (AWK13.14) and coverage retuning (AWK13.15).

## Assumptions and blockers

| Type | Item | Evidence or resolution path |

| Assumption | The supervisor refactor is structural; existing tests cover the behavior. | Round-2 supervisor/completion-gate tests pass at HEAD. |
