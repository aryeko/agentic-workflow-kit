## Scope boundary

**In scope**

- Define incremental pre-PR reviewer continuity semantics.
- Prefer reusing the same review subagent/thread for follow-up loops when the host can continue it.
- Define the fallback when host continuation is unavailable: spawn a new read-only review subagent
  with the incremental packet and do not treat that as a downgrade.
- Journal continuity explicitly with `loop`, `agentId`, `previousAgentId`, and
  `continuityMode`.
- Preserve fail-closed behavior for `implement.review.prePr.mode: subagent`: a real subagent
  review is still required; only same-thread reuse is best effort.

**Out of scope**

- New public streaming surface; AWK09 owns it.
- External GitHub/Codex PR review reaction parsing; AWK11 owns it.
- Hosted review thread persistence beyond what the host tool exposes.
- Adding a config key that requires same-thread reuse.

## Artifact boundaries

PRD owns product outcomes. Technical solution owns runtime-event and prompt-boundary architecture.
Tracker owns sequencing. This story brief owns local review-continuity scope. The detailed technical
story spec must define exact event fields, analyzer behavior, and any instruction changes.
