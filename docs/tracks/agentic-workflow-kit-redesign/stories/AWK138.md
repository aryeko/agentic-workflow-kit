## Scope boundary

**In scope**

- Route `codex_reply` / `codex_interrupt` MCP tools through `StoryRunner.controlChild` / `abort`
  instead of the standalone path in `mcp/codexControl.ts`; keep tool names as back-compat aliases.
- Add a driver factory/registry keyed by `OrchestratorDriver`; instantiate `CodexMcpStoryRunner` in
  one place; resolve the driver from the registry at the current ~4 call sites.
- Resolve the artifact root from the driver / a single neutral constant instead of hardcoding
  `.codex/...` in the neutral resolved config; keep the existing path as default.
- Generalize the provider-named `ResolvedWorkflowConfig.codex` field via a neutral alias; keep
  `codex` readable for back-compat.

**Out of scope**

- Renaming config keys, artifact directories, CLI/MCP tool names, or the result envelope.
- Adding a real second driver (this story only proves the seam is clean).
- Prompt-rendering relocation (already handled in round 1 / AWK13.1).

## Assumptions and blockers

| Type | Item | Evidence or resolution path |
