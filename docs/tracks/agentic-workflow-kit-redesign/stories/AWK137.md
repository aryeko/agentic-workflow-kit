## Scope boundary

**In scope**

- Update root `CHANGELOG.md` to reflect the 0.5.x reality (or point to the package changelog as the
  source of truth); fix `SECURITY.md` supported-versions to `0.5.x`.
- Make `docs/getting-started.md` command forms consistent with the repo-checkout context it assumes
  (no bare `agentic-workflow-kit …` where the guide otherwise uses `pnpm agentic-workflow-kit -- …`).
- Add `engines: { node: ">=24" }` to the published `packages/orchestrator/package.json`.
- Align the `workflow-autopilot` SKILL tool list with the shipped tools (`watch_run_start/poll/stop`,
  child controls, `workflow_*` facade), and re-materialize the plugin mirror.
- Document the AWK13.4 approval default and AWK13.2 verification behavior where adopters read setup
  and safety guidance.

**Out of scope**

- The release changeset and version bump (AWK14 owns them).
- Rewriting canonical docs already consolidated by AWK13 beyond the stale facts named here.
