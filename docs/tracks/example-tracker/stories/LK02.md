# LK02 story brief

not implementation-ready; create a detailed technical story spec before plan/code

## PRD criteria

| Criterion | Product outcome |
| --- | --- |
| L-2 | A short code 301-redirects to the original URL. |

## Technical solution sections

| Section | Relevance |
| --- | --- |
| Not required for this simple example | Redirect can follow the boundary established by LK01. |

## Dependencies

| Dependency | Reason |
| --- | --- |
| LK01 | Needs a short-code store before lookup/redirect can exist. |

## Scope boundary

**In scope**

- Resolve short code to original URL.
- Return a permanent redirect for known codes.

**Out of scope**

- Click counting; see LK03.
- Link creation; handled by LK01.

## Candidate surfaces

- **Files/modules:** redirect route/handler, link store lookup.
- **Queries/schema:** read by short code.
- **Prompts/tools:** none expected.
- **Events/metrics:** optional redirect audit event.
- **Components/routes:** short-code redirect route.

## Validation expectations

- Known short codes redirect with 301.
- Unknown short codes return a bounded not-found response.

## Open technical questions

| Question | Blocking? | Resolution expected in detailed spec |
| --- | --- | --- |
| What exact not-found response shape should unknown codes use? | yes | Decide status, body, and user-facing fallback. |
