# Glossary

| Term | Definition |
| --- | --- |
| Agentic coding tool | A coding agent host such as Codex or Claude that can reason, edit files, run tools, and report results. |
| Autopilot | Runtime mode that dispatches eligible tracker stories under configured policy until complete or stopped. |
| Budget policy | Configured limits and actions for wall time, tokens, tool calls, failures, or cost. |
| Child session | Agent session launched by the runtime for bounded story or support work. |
| Contract-backed track | Markdown backlog that conforms to the kit tracker schema and can be read by automation. |
| HLD | High-level design; in this product, the technical solution artifact for complex work. |
| Local-first | Product mode where repo files and local artifacts are sufficient; no hosted backend is required. |
| PRD | Product requirements document that owns what and why before technical decomposition. |
| Recoverable stopped state | A run outcome that stops safely and includes enough evidence for a user to resume, retry, salvage, or abandon. |
| Runtime | The CLI/MCP orchestrator that launches and supervises agent sessions against tracker stories. |
| Story-level autonomy | Running the implementation loop for exactly one tracker story. |
| Track-level autonomy | Running the implementation loop across eligible stories in a tracker. |
